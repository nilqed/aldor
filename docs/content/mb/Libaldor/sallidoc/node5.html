How do I use SALLI in my programs?

Once SALLI is properly built, you need to set the following environment variables before using it:
  • the variable SALLIROOT should be set to the main salli directory, or to salli/debug if you want to use the debug version of the library;
  • $SALLIROOT/include should be appended to your INCPATH variable;
  • $SALLIROOT/lib should be appended to your LIBPATH variable;
In your ALDOR programs, use #include "salli" instead of #include "axllib". When building your final executable, add the options
-lsalli -y$SALLIROOT/lib
to your compiler command line. If you intend to make use of the GMPInteger or GMPFloat type, make sure to add the option
-cruntime=foam-gmp3,gmp-3
to your compiler command line. This will ensure that GMP makes use of the ALDOR garbage collector. Note that the GMP 3.0 or 3.1 library should then be in a file called libgmp-3.a. If you are running SALLI inside the compiler interactive loop, then add the line #include "sallinterp" after #include "salli", which will import various things for interactive use and make the interpreter loop print values automatically. Note that GMPInteger and GMPFloat are not available in the interactive loop. As with any ALDOR program, do not forget the -q3 option in order to optimize your programs, specially if performance is an issue.

Before using SALLI for the first time, please check your installation by running make in the salli/test directory, followed by running testall.

Please report any installation problem or bugs you encounter to salli@sophia.inria.fr.
