Arithmetic

SALLI contains 3 sorts of integers: MachineInteger provides the full-word signed machine integers, while AldorInteger and GMPInteger both provide ``infinite'' precision software integers. GMPInteger is an interface to the GMP 3.1 package and requires libgmp.a 3.0 or 3.1 to run, while the actual implementation of AldorInteger can be selected at link-time to be either the built-in software integers or also the GMP package. All 3 are of category IntegerType, so exactly the same functions can be applied to them. Using GMPInteger will generally produce more efficient programs if you are computing with large integers. In addition, it gives you access to the the in-place operations of GMP, which are not available with AldorInteger. On the other hand, GMPInteger cannot be used from inside the compiler interactive loop. In addition, MachineInteger exports 3 useful constants: min and max, whose values are the smallest and greatest machine integers, and bytes, whose values is the machine wordsize in bytes. The bytes constant allows you to write wordsize-dependent code, for example when generating half-word-size prime numbers. A MachineInteger value n can be coerced to either an AldorInteger or GMPInteger via n::AldorInteger or n::GMPInteger, and the reverse conversion is provided via the machine function, i.e.n := machine m. This last conversion can of course lose precision if m does not fit within a machine word. The general form for iterating over a range of integers is
for variable in from..to by step | condition repeat { ... }
where the to parameter and the by and ``| condition'' parts are optional.

Besides the integers, SALLI also provides 3 sorts of floating point numbers: single-precision (SingleFloat), double-precision (DoubleFloat) and ``infinite'' precision software floats (GMPFloat), the latter being an interface to GMP 3.1. All 3 are of category FloatType, which provides only a limited set of standard functions.
